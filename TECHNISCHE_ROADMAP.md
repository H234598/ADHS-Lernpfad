# Technische Roadmap

Diese Roadmap ergänzt die fachliche ROADMAP.md. Die fachliche Lernpfad-Roadmap bleibt unverändert.

Sie beschreibt ausschließlich die technische Weiterentwicklung der Plattform.

## Zielbild

Der ADHS-Lernpfad soll langfristig eine wissenschaftlich gepflegte, automatisierte und reproduzierbare Wissensplattform werden.

Schwerpunkte:

- reproduzierbare Builds
- sichere Automatisierung
- robuste Veröffentlichung
- nachvollziehbare Qualitätssicherung
- langfristige Wartbarkeit

## Phase 1 – Governance-Basis

- [ ] technische Milestones einführen
- [ ] Issues systematisch klassifizieren
- [ ] Labels und Zuständigkeiten standardisieren
- [ ] Issue- und Pull-Request-Templates etablieren
- [ ] Definition of Done dokumentieren

## Phase 2 – Automation Recovery und Observability

- [ ] Run-ID-System
- [ ] atomare Statusdateien
- [ ] Laufhistorie
- [ ] verbesserte Fehlerdiagnose
- [ ] Recovery-Prozesse
- [ ] automatische Diagnoseberichte

## Phase 3 – Wissensgraph und Infrastruktur

- [ ] Wissensgraph-Generator stabilisieren
- [ ] Graph-Health-Checks
- [ ] fehlende Nodes erkennen
- [ ] defekte Links markieren
- [ ] Webdarstellung verbessern

## Phase 4 – Release Management

- [ ] GitHub Releases standardisieren
- [ ] Release Notes automatisieren
- [ ] Artefakte versionieren
- [ ] Prüfsummen veröffentlichen
- [ ] Rollback-Prozess dokumentieren

## Phase 5 – Security Hardening

- [ ] SECURITY.md
- [ ] Dependency Monitoring
- [ ] Workflow-Berechtigungen prüfen
- [ ] Supply-Chain-Sicherheit verbessern
- [ ] Secrets Audit

## Phase 6 – Wissenschaftliche Qualitätsinfrastruktur

- [ ] Literatur-Metadaten validieren
- [ ] Review-Zyklen verwalten
- [ ] Evidenz-Level prüfen
- [ ] Aktualität von Kapiteln überwachen

## Phase 7 – Feedback und Qualitätsmanagement

- [ ] strukturierte Feedbackkanäle
- [ ] Bug- und Verbesserungstracking
- [ ] wissenschaftliche Korrekturprozesse

## Phase 8 – Plattformmetriken

Technische Metriken:

- Build Success Rate
- Automation MTTR
- Graph Health Score
- Release Completeness
- Dependency Status

Inhaltliche Qualitätsmetriken:

- Quellenqualität
- Evidenzabdeckung
- Reviewstatus
- Aktualität

## Grundsatz

Die technische Roadmap erweitert die Wartungs- und Betriebsdokumentation. Sie ersetzt nicht die fachliche Roadmap des Lernpfades.

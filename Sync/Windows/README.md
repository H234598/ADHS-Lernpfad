# Windows-Synchronisierung

Die Ordnerstruktur ist vorbereitet; ein produktives Windows-Paket wird später in einem eigenen, systemübergreifenden Plan ausgearbeitet.

## Vorgesehene Varianten

- PowerShell-Skript für manuelle Ausführung
- Aufgabenplanung für regelmäßige Läufe
- Auswahl zwischen `safe-pull`, `prompt-pull`, `forced-pull` und einem späteren `full-sync`
- getrennte Behandlung von Obsidian- und Syncthing-Metadaten

Die gemeinsame Modusdefinition steht unter [[Sync/MODES|Synchronisationsmodi]].

> [!info]
> In diesem Ordner liegt absichtlich noch kein ausführbares Skript. Eine ungetestete Teilimplementierung wäre riskanter als ein klar gekennzeichneter Platzhalter.
